## 1. 核心命题

> **专业服务是最后一个还没有被工厂化的主要行业。AI 是那个工厂。**

历史上每次生产力革命，本质是把"手工艺"变成"工厂"：
- 工业革命：物理劳动 → 机器制造
- 互联网：信息传播 → 算法分发
- **AI 时代：认知工作 → 智能工厂**

专业服务（billing、marketing、legal、accounting、HR、ops）至今仍是手工艺模式：贵、慢、不可扩展、知识锁在个人脑中。

**这是 $500B+ 的被压抑的需求。**

100,000 家中小公司一直需要 McKinsey 级别的 marketing、一流律所级别的合规、专业 agency 级别的 GTM。他们负担不起。现在可以了。

## 2. 为什么现在

两条曲线，方向相反：

```
认知工作成本：  ████████░░░░░░  → 趋近于零（模型推理成本每年 -10x）
认知工作需求：  ░░░░░░████████  → 指数增长（复杂度、速度、全球竞争）
```

这个剪刀差创造了历史上最大的供需错配之一。

**传统专业服务的根本局限：**
每个 engagement 的学习锁在人脑里。顾问离职，知识消失。规模化 = 雇更多人。没有复利。

**AI-native 模式的结构性优势：**
每个 engagement 教会系统。系统永远记住。规模化 = 系统更聪明。有复利。

## 3. 战略模型：AI-native Berkshire

**不是 SaaS 公司。不是咨询公司。不是 PE fund（不以退出为目的）。**

而是：**用 AI engine 持续创建并永久持有 AI-native 专业服务公司的集团。**

```
Berkshire（传统）：资本分配能力 → 收购优质现有企业 → 永久持有 → 复利
OpenVibe 模式：  AI engine    → 创建 AI-native 企业 → 永久持有 → 复利
```

唯一的区别：Berkshire 用资本收购，OpenVibe 用 AI engine 创建。

### 集团结构

```
OpenVibe Engine（核心，永远不出售）
│  - AI orchestration + workflow
│  - 中央 GTM engine（系统性获客）
│  - 跨公司 playbook 积累
│  - 数据飞轮
│
├── 自营层（15-20 家，每个主要 domain 一家）
│     每家服务 3,000-8,000 终端客户
│     永久持有，65-70% 利润
│
└── 平台层（O5，外部 founders 建细分领域公司）
      1000+ 家，每家服务 100-500 客户
      OpenVibe 收 revenue 的 10%
```

**自营 + 平台合计：服务 ~100,000 家终端客户**

## 4. 为什么是多家公司而不是一家

AI engine 可以从单一实体服务 10 万客户——技术上完全可行。分拆成多家公司的原因不是 AI 能力，而是：

| 原因 | 解释 |
|------|------|
| **信任是领域特定的** | 客户信任"dental billing specialist"，不信任"AI 综合服务" |
| **分发渠道是领域特定的** | 每个行业有自己的协会、转介绍网络，独立品牌才能参与 |
| **监管边界** | HIPAA、律所规定、CPA 牌照——独立法律实体是最干净的隔离 |
| **人才层不可互换** | 做牙科 billing 的 2-3 个专家 ≠ 做 construction marketing 的专家 |

**自营层：15-20 家**（每个主要 domain 一家，如 healthcare billing、B2B marketing、legal research...）
**平台层：1000+**（细分领域由外部 founders 建，如牙科 billing / 医院 billing / 诊所 billing 分三家）

## 5. 四层战略（O4+O6+O12+O5）

这四个机会不是平行选项，是垂直层级，互相支撑：

```
O6  判断力放大器     ← 技术层：AI engine，让 1 人服务 100 个客户
O4  专业服务解绑     ← 人才层：离开大公司的 domain experts 提供判断力
O12 持有型工厂       ← 资本层：自己建并永久持有自营公司
O5  平台型工厂       ← 规模层：开放 engine 给外部 founders 建长尾
```

**各层如何互相需要：**
- O6 是基础：没有引擎，其他三层都不成立
- O4 给 O12 提供燃料：自营公司的人类判断力层来自从大公司出来的专家
- O12 给 O5 提供 proof point："我们自己做到了，你也可以"
- O5 让 O12 无法覆盖的细分领域被平台上的外部 founders 覆盖

## 6. 复利机制

这个模型有三层复利，叠加在一起：

**Layer 1：引擎复利**
每个 engagement → 系统学到更好的解法 → 下一个 engagement 更快更好

**Layer 2：资本复利**
老公司的利润 → 建新公司 → 无需外部融资，自我强化

**Layer 3：数据复利**
跨 domain 积累的"认知工作实际怎么完成"的数据 → 比任何单一公司都深的 intelligence → 无法复制的 moat

## 7. 规模数学

```
自营层（15-20 家）：
  平均每家 5,000 客户 × $5K/月 × 65% margin
  = $195M operating profit

平台层（1,000 家外部公司）：
  平均每家 $2M revenue × 10% take rate
  = $200M revenue

总计（成熟阶段）：
  ~$400M+ operating profit/年
  不含引擎本身的估值溢价
```

**Vibe 是第一家 AI-native 公司，也是整个模型的 Phase 0。**

Vibe dogfood 不是"帮 Vibe 卖更多硬件"。是在回答三个必须为真的前提：

| 前提 | Vibe dogfood 验证什么 |
|------|---------------------|
| AI 交付质量达标 | 客户愿意付费、续费 |
| 工厂可以复制 | 第 1 家公司的 playbook 可以建第 2、10、100 家 |
| 系统性获客可行 | GTM engine 可以规模化，不依赖人际关系 |

**Vibe 的 40K 客户 = 自营公司 Phase 1 的第一批分发渠道。**
已经买了 board 的公司，是最自然的 "Vibe AI Marketing" 或 "Vibe AI Ops" 的第一批客户。

## 10. 平台扩展机制

### 设计原则

> **平台的核心产品不是基础设施，是积累的 intelligence。**

Shopify 卖工具。Bloomberg 卖数据网络，工具只是访问入口。OpenVibe 平台是后者——domain experts 付费访问的是跨所有 firm 积累的 intelligence，不是软件本身。

### 三层结构

```
Tier 1：自营层（15-20 家）
  OpenVibe 全资持有，最高标准，最深数据
  品牌背书："这是我们自己跑的"

Tier 2：认证伙伴层（100-200 家）
  审核准入 + 小比例股权
  共享品牌 + 全套 engine
  Revenue share 10% + 数据贡献义务

Tier 3：开放平台层（1000+ 家）
  自助入驻，sub-brand（"Powered by OpenVibe"）
  数据贡献换取 intelligence 访问权
```

三层创造向上流动的动力：Tier 3 → Tier 2 → Tier 1 的升级条件 = 数据贡献质量 + 客户满意度。

### 核心机制：强制数据贡献 + 价值回报

```
新 firm 加入：
  贡献自己的数据（少）→ 获得平台积累的所有数据（多）
  第一天就比独立运营更聪明

老 firm 离开：
  带走自己的数据 → 失去所有其他 firm 的积累
  立刻退化回孤立状态
```

**Switching cost = 失去整个 intelligence 网络。** 不是合同锁定，是价值锁定。越晚加入，竞争优势越小。

### 飞轮

```
更多 firm 加入
  → 更多 domain 数据
  → Platform intelligence 更深
  → 每个 firm 服务质量提高
  → 更多客户 → 更多数据
  → 吸引更多 domain experts
  → 更多 firm
```

### 获客冷启动：中央 GTM 引擎

平台上每家 firm 最大的问题是找第一批客户。解法：

- **Vibe 40K 客户分发**：已买 board 的公司自然转介给相关 firm
- **跨 firm 互相转介绍**：AI billing firm 的客户 → 推荐给 AI marketing firm
- **平台统一品牌 inbound**："OpenVibe certified firm" 成为质量信号，客户主动搜索

**平台 vs 独立运营的最大差距**：独立 firm 自己解决获客；平台上，获客是共享基础设施。

### 质量治理

用结果数据治理，不用人工审核：

```
每个 engagement → 客户打分 + 结果数据
  → 低于 threshold：警告 → 降级 → 移除
  → 高于 threshold：升级 → 更多获客分发
```

质量是动态的，坏的 firm 自动淘汰，好的 firm 自动放大。

### 历史类比

| 类比 | 借鉴什么 |
|------|---------|
| Visa / Mastercard | 网络效应：更多 firm → 更多客户信任 → 更多 firm |
| Michelin Guide | 认证即价值：被选入 = 质量信号，无需自建品牌 |
| Bloomberg Terminal | 数据网络 > 工具本身：付费访问的是 intelligence |

OpenVibe 比这三个多一件事：**intelligence 通过所有 firm 的真实工作自动积累，无需主动维护。**

## 11. 公司边界理论

### 为什么 OpenAI / Anthropic 不直接做这个

表面上他们有最好的 AI，为什么不 own 1000 家专业服务公司？

1. **利益冲突**：他们的客户就是这些公司。自己参与竞争 = 和客户抢生意。AWS 不做 SaaS 应用，原因相同。
2. **能力缺口**：训练模型 ≠ 知道"好的牙科 billing 长什么样"。技术能力和 domain judgment 是两件事。
3. **使命与监管**：Anthropic 是 PBC，运营 1000 家专业服务公司会立刻面临垄断指控和使命稀释。

### 公司边界的新定义

传统理论（Coase）：公司边界由内外协调成本决定。

AI 之后内外协调成本都趋近于零，新的边界决定因素是：

> **公司是 judgment 的容器，不是执行的容器。**

公司边界 = 一套一致的 domain judgment 的边界，包含三件事：
- **判断力**：谁决定"什么是好结果"
- **问责**：出了问题谁负责
- **信任积累**：关系是跟谁建立的，品牌代表什么承诺

### 扩展边界的系统

```
第一层：AI engine 泛化
  更多 domain judgment 从数据里学，减少对人类 expert 的依赖

第二层：Platform（O5）
  把 first-order domain judgment 外包给 domain experts
  OpenVibe 只需 hold 一件事：
  "什么是好的 AI-native 专业服务公司"（meta-judgment）

第三层：收购 judgment
  收购已有 domain judgment 的人或小公司
  judgment + engine 结合，快于从零培养
```

### 最终边界

| 层次 | 边界类型 | 可突破？ |
|------|---------|---------|
| 监管、资本、时间 | 表层 bound | 是，工程问题 |
| 问责必须具身化（手术、法庭辩护） | 中层 bound | 很难，社会/法律设计 |
| **"你在回答什么问题"失去 coherence** | **深层 bound** | **这才是真正的边界** |

你被 bound 住的那一天，不是因为监管或资本，而是因为**不再清楚自己在 optimize 什么**。当前的问题足够清晰：*如何把专业服务交付的成本降到基础设施级别。* 这个问题支撑整个 portfolio 的 coherence。

## 12. 关键假设与 Kill Signals

### 必须为真的假设

1. **AI 交付质量**：专业服务的核心工作可以被 AI 完成到客户愿意付费的水准
2. **工厂可复制性**：Domain A 的 playbook 提炼后可以加速 Domain B 的建立
3. **系统性获客**：客户获取可以被 engineered，不依赖个人关系
4. **专家供给**：足够多的 domain experts 愿意成为 AI-native firm 的判断力层

### Kill Signals

- Phase 0 M3：AI 输出质量被 Vibe 团队评为"不如自己做"→ 引擎前提不成立
- Phase 0 M6：获客无法系统化，仍依赖 Charles 个人关系 → 工厂前提不成立
- Phase 1：第一家非 Vibe 自营公司无法在 3 个月内获得第一个付费客户 → 跨 domain 迁移失败

## 13. 竞争位置

**为什么别人不做这个？**

| 可能的竞争者 | 为什么不会赢 |
|------------|------------|
| McKinsey / Accenture | 商业模式是人力，转型意味着自我颠覆 |
| 纯 SaaS 公司 | 卖工具，不对结果负责，无法建立 track record |
| 垂直 AI 创业公司 | 单一 domain，无法建立跨 domain 的引擎复利 |
| Anthropic / OpenAI | 卖基础模型，战略上不做下游服务 |

**真正的威胁：** 一家有足够资本和执行力的公司，在 OpenVibe 验证模型之前，用同样的思路快速复制。

**时间窗口：** 18-24 个月。这是 Phase 0-1 的紧迫性来源。
